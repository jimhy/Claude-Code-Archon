# 用户画像（全局）

> **部署位置**：`~/.claude/user-profile.md`（全局，跨项目共享）
>
> **用途**：所有 agent 在与用户直接沟通前必读。按画像调整：
> - 术语深度（技术 vs 业务语言）
> - 问题详细度（技术选型 vs 使用场景）
> - 决策权重（AI 代决定 vs 给选项 vs 开放讨论）
>
> **维护**：kickoff-agent 首次建立；后续 agent 发现画像不准可更新。用户也可手动编辑。

## 沟通模板（按画像）

### 小白模板

**术语规则**：
- ❌ "我帮你做一个 React SPA，部署到 Vercel"
- ✅ "我帮你做一个网页应用，你可以在电脑和手机浏览器里打开"

**问题风格**：
- 问使用场景，不问技术
- 用生活化类比（"像微信那样的聊天列表"而非"用 virtualized list"）
- 问"你希望看到什么"而非"你希望怎么实现"

**反馈风格**：
- 少给细节，多给"做好了，你打开这个链接试试"
- 截图/录屏 > 文字描述
- 失败时说"这个我搞不定，给你三个选择：A/B/C，你选哪个？"

### 非技术（AI 老手）模板

**术语规则**：
- ❌ "用 Next.js + tRPC + Prisma"
- ✅ "用一个现代 Web 框架 + 数据库，保证打开快、数据安全"

**问题风格**：
- 可提产品类比（"类似 Notion"、"类似 Trello"）
- 解释关键技术选择的**影响**，不讲实现（"选 Web 意味着 iPhone/Android 都能用但没有推送"）
- 让用户拍板**产品**决策，技术交给 AI

### 普通技术模板

**术语规则**：
- 正常技术术语
- 新术语要解释（如果偏门）

**问题风格**：
- 技术栈偏好问清楚
- 给 2-3 个选项 + 推荐一个 + 说明理由
- 承认不熟悉时用户可纠正（"我看你没做过 Flutter，建议用 React Native，或你坚持 Flutter 也可以"）

**决策风格**：
- 给选项 + 推荐
- 解释权衡（"A 方案快但锁死 Vercel，B 方案慢一点但更灵活"）

### 专家技术模板

**术语规则**：
- 全套技术术语，无需解释
- 直接讨论架构、权衡、边界情况

**问题风格**：
- 开放式问题（"你怎么看 RSC vs SPA"）
- 不给"推荐"，给"各自 trade-off"
- 允许用户推翻 AI 方案

**决策风格**：
- 用户定方向，AI 执行 + 提醒盲点
- 不要"我建议这样更好"除非有非常具体理由

## 画像更新触发条件

### 升级画像
用户表现出比初判更高水平：
- 小白 → 非技术：用户主动提"像 Notion 那样" / "需要后端" 等产品级术语
- 非技术 → 普通技术：用户说 "用 React" / "API 返回 JSON"
- 普通 → 专家：用户讨论"锁服务级 vs 应用级" / "CAP 定理" 等架构术语

### 降级画像
用户表现出比初判更低水平：
- 技术 → 非技术：用户问"这是什么意思"过多技术基础词
- 非技术 → 小白：用户不理解"链接"、"浏览器"等基础概念

### 用户主动纠正
- "别把我当小白" → 升一级
- "你说的我听不懂" → 降一级
- "我是 XX 工程师" → 按职位调整

**更新时**：
1. 修改本文件对应字段
2. 在"元信息"增加更新记录
3. 通知当前所有活跃 agent 新画像

## 示例：常见画像组合

### 案例 1：Vibe Coder（非技术但会用 AI）
- 技术层级：AI 老手非技术
- AI 经验：Cursor 用户
- 项目经验：玩具项目几个
- 技术栈：N/A（不写代码，让 AI 写）
- 沟通偏好：你直接决定 / 随意 / 能立刻看结果

→ kickoff-agent 用"非技术（AI 老手）模板"；技术选型 AI 全决定；只在关键产品决策让用户拍板。

### 案例 2：全栈工程师
- 技术层级：专家技术
- AI 经验：Claude Code 深度用户
- 项目经验：大型项目
- 技术栈：精通 TS/React/Node/Postgres；了解 Go/Python；不熟 iOS 原生
- 沟通偏好：讨论一下 / 正式 / 每天检查

→ kickoff-agent 用"专家技术模板"；架构一起讨论；技术栈按用户熟悉度优先。

### 案例 3：产品经理转做 side project
- 技术层级：非技术
- AI 经验：ChatGPT 常用户
- 项目经验：做过产品（不是工程）
- 技术栈：N/A
- 沟通偏好：给我选项 / 随意 / 每天检查

→ 产品决策深入讨论（PM 擅长），技术决策 AI 全权 + 告知权衡。
